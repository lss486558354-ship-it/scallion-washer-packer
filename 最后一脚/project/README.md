## 1. 系统概述

本系统采用 STM32F103VE 作为主控制器，香橙派5 MAX 作为算力单元（视觉识别），实现小葱的自动切割、清洗、打包功能。

### 系统架构

```
┌─────────────────┐      串口通信(115200)      ┌─────────────────┐
│   香橙派5 MAX   │ ◄────────────────────────► │  STM32F103VE    │
│   (视觉+AI)    │                             │   (运动控制)    │
└─────────────────┘                             └────────┬────────┘
                                                        │
                                    ┌───────────────────┼───────────────────┐
                                    │                   │                   │
                                    ▼                   ▼                   ▼
                              ┌──────────┐        ┌──────────┐        ┌──────────┐
                              │ 切割电机 │        │传送带电机│        │ 打包电机 │
                              │(步进电机)│        │(步进电机)│        │(步进电机)│
                              └──────────┘        └──────────┘        └──────────┘
```

## 5. 快速开始

### 5.1 STM32 工程配置

1. **使用Keil MDK打开工程**
   - 项目路径：`Project/RVMDK（uv5）/BH-F103.uvprojx`

2. **添加新的驱动文件到工程**
   - 在Keil中右键 `User` 分组
   - 添加以下文件：
     ```
     User/protocol/protocol.c
     User/motor/bsp_motor.c
     User/servo/bsp_servo.c
     User/sensor/bsp_sensor.c
     User/limit_switch/bsp_limit_switch.c
     ```

3. **包含头文件路径**
   - 添加：`User/protocol`, `User/motor`, `User/servo`, `User/sensor`, `User/limit_switch`

4. **编译并下载**

### 5.2 香橙派配置

1. **安装Python依赖**
   ```bash
   pip3 install pyserial
   ```

2. **连接串口**
   - 确认硬件连接
   - 测试通信：
     ```bash
     python3 stm32_comm.py
     ```

3. **查看日志输出**
   - 应显示 "串口连接成功"
   - 心跳包正常发送

## 6. 通信协议

### 6.1 帧格式

```
┌──────┬──────┬──────┬─────────┬──────┐
│ 帧头  │ 命令  │ 长度  │  数据   │ 校验  │
│ 0xAA │ CMD  │ LEN  │ DATA[] │  CRC  │
└──────┴──────┴──────┴─────────┴──────┘
```

- **帧头**：0xAA（固定）
- **命令**：1字节，命令类型
- **长度**：1字节，数据区长度
- **数据**：N字节，有效载荷
- **校验**：1字节，异或校验

### 6.2 命令列表

#### STM32 → 香橙派

| 命令 | 值 | 数据格式 | 说明 |
|------|------|---------|------|
| CMD_SENSOR_DATA | 0x01 | 见下方 | 传感器数据 |
| CMD_MOTOR_STATUS | 0x02 | 见下方 | 电机状态 |
| CMD_LIMIT_SWITCH | 0x03 | status:1B | 限位开关状态 |
| CMD_SYSTEM_STATUS | 0x04 | status:1B, error:1B | 系统状态 |
| CMD_TASK_COMPLETE | 0x05 | task_id:1B | 任务完成 |
| CMD_ERROR_REPORT | 0x06 | error:1B | 错误报告 |

**传感器数据格式 (CMD_SENSOR_DATA)**：
```
字节0-1:   电机1位置
字节2-3:   电机2位置
字节4-5:   电机3位置
字节6-7:   电机1速度
字节8-9:   电机2速度
字节10-11: 电机3速度
字节12:    ADC采样值
```

#### 香橙派 → STM32

| 命令 | 值 | 数据格式 | 说明 |
|------|------|---------|------|
| CMD_START_TASK | 0x10 | - | 开始任务 |
| CMD_STOP_TASK | 0x11 | - | 停止任务 |
| CMD_PAUSE_TASK | 0x12 | - | 暂停任务 |
| CMD_RESUME_TASK | 0x13 | - | 恢复任务 |
| CMD_SET_MOTOR_SPEED | 0x14 | id:1B, speed:2B | 设置电机速度 |
| CMD_SET_MOTOR_POS | 0x15 | id:1B, pos:2B, mode:1B | 设置电机位置 |
| CMD_SET_SERVO_ANGLE | 0x16 | id:1B, angle:2B | 设置舵机角度 |
| CMD_RESET_SYSTEM | 0x17 | - | 系统复位 |

## 7. API参考

### 7.1 电机控制

```c
// 初始化
Motor_Init();

// 设置速度（有符号，正转/反转）
Motor_SetSpeed(MOTOR_1, 3000);    // 电机1 正转
Motor_SetSpeed(MOTOR_1, -3000);   // 电机1 反转

// 设置目标位置（步进电机）
Motor_MoveToPosition(MOTOR_2, 5000);

// 停止电机
Motor_Stop(MOTOR_1);

// 获取当前位置
int32_t pos = Motor_GetPosition(MOTOR_1);
```

### 7.2 舵机控制

```c
// 初始化
Servo_Init();

// 设置角度（0-180度）
Servo_SetAngle(SERVO_1, 90.0f);   // 舵机1转到90度
Servo_SetAngle(SERVO_2, 45.0f);   // 舵机2转到45度

// 获取当前角度
float angle = Servo_GetAngle(SERVO_1);
```

### 7.3 传感器读取

```c
// 初始化
Sensor_Init();

// 读取传感器值
float pressure = Sensor_GetValue(SENSOR_PRESSURE);
float water_level = Sensor_GetValue(SENSOR_WATER_LEVEL);
float weight = Sensor_GetValue(SENSOR_WEIGHT);

// 校准
Sensor_AutoZero(SENSOR_PRESSURE);  // 自动归零
```

### 7.4 限位开关

```c
// 初始化
LimitSwitch_Init();

// 读取状态
if (LimitSwitch_IsTriggered(LIMIT_M1_ORIGIN)) {
    // 电机1已到达原点
}

// 读取所有限位状态
uint8_t status = LimitSwitch_ReadAll();
```

### 7.5 香橙派通信

```python
# 连接STM32
stm32 = STM32Communicator('/dev/ttyS4', 115200)
stm32.connect()

# 发送指令
stm32.send_motor_speed(2, 1000)      # 设置电机2速度
stm32.send_servo_angle(1, 45)        # 设置舵机1角度

# 接收回调
def on_sensor(data):
    print(f"电机位置: {data['motor1_position']}")

stm32.sensor_callback = on_sensor
```

## 8. 常见问题

### Q1: 串口通信失败
**检查项**：
1. 串口线连接是否正确（TX-RX交叉）
2. 波特率是否匹配（115200）
3. 是否有电平转换（如需要）

### Q2: 电机不转
**检查项**：
1. 驱动器电源是否正常（24V）
2. 使能信号是否正确（EN低电平使能）
3. PWM输出是否正常（用示波器检查）
4. 方向信号是否正确

### Q3: 限位开关一直触发
**检查项**：
1. 接线是否正确
2. 触发电平设置是否正确（默认低电平触发）
3. 开关类型（常开/常闭）

### Q4: 舵机抖动
**可能原因**：
1. 电源电压不足（需要5-6V）
2. PWM频率不对（50Hz）
3. 信号线接触不良
