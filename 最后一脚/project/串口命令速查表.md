## 一、帧格式

```
┌────────┬────────┬────────┬─────────────┬────────┐
│ 帧头   │ 命令   │ 长度   │ 数据         │ 校验   │
│ 1字节   │ 1字节  │ 1字节  │ N字节        │ 1字节  │
├────────┼────────┼────────┼─────────────┼────────┤
│  0xAA  │  CMD   │  len   │ data[0..N-1] │  CS   │
└────────┴────────┴────────┴─────────────┴────────┘
```

**校验和计算：**

```
CS = CMD ^ len ^ data[0] ^ data[1] ^ ... ^ data[len-1]
```

**帧总长度：** 4 + len（帧头+命令+长度+校验 = 4，加数据长度）

## 四、计算公式速查

### 4.1 步进电机速度转换

**目标：** 将速度 Hz 值转换为高/低字节

```
speed_hz = 2000
speed_h  = speed_hz >> 8         = 0x07
speed_l  = speed_hz & 0xFF      = 0xD0

帧: [ID] [speed_h] [speed_l]
示例: 01    07       D0
```

**反向计算：**

```
speed_hz = (speed_h << 8) | speed_l
```

### 4.2 通用速度+方向命令值转换

**步进电机（类型=0x01）：值 = 速度Hz**

```
值 = 速度Hz
示例：值=2000 → speed_h = 0x07, speed_l = 0xD0
```

**直流电机（类型=0x02）：值 = 占空比%**

```
值 = 占空比(0~100)
示例：值=80 → speed_h = 0x00, speed_l = 0x50
```

### 4.3 偏移量计算（有符号16位）

**int16 → 高/低字节（struct.pack('<h', offset)）**

```
偏移量 = +100:
  int16 = 0x0064
  H = 0x00, L = 0x64

偏移量 = -100:
  int16 = 0xFF9C
  H = 0xFF, L = 0x9C

偏移量 = +500:
  int16 = 0x01F4
  H = 0x01, L = 0xF4
```

**反向（有符号恢复）：**

```
if H >= 0x80:
    offset = (H << 8 | L) - 65536
else:
    offset = (H << 8 | L)
```

### 4.4 超声波测距计算

**HC-SR04 距离公式：**

```
echo_high_us = SysTick差值 / 72   （72MHz主频）
distance_cm  = echo_high_us / 58
distance_mm  = echo_high_us * 10 / 58
            ≈ echo_high_us / 5.8

或（使用乘除合并）：
distance_mm  = (diff * 17) / 72000
  其中 diff = SysTick计数值差（72MHz主频）
```

**实际距离示例：**

```
echo_high_us = 2900μs:
  distance_cm = 2900 / 58 = 50cm
  distance_mm = 50 * 10 = 500mm

echo_high_us = 580μs:
  distance_mm = 580 * 10 / 58 = 100mm
```

### 4.5 丝杆目标位置计算

```
SCREW_POS_K = 50   （每mm对应50步，需标定）
HEIGHT_REF_MM = 200 （参考高度，丝杆=0时对应高度）

target_pos = (HEIGHT_REF_MM - 实测高度) × SCREW_POS_K

示例：
  实测高度 = 150mm → target_pos = (200-150) × 50 = 2500步
  实测高度 = 180mm → target_pos = (200-180) × 50 = 1000步
  实测高度 = 200mm → target_pos = 0
```

### 4.6 丝杆软件限位计算

软件限位通过定时器实现（第一道保护），硬件限位开关兜底（最后保险）。

**丝杆运动时序：**

```
进入 PROCESS_SCREW_ADJUST：
  STEP1: 正转下降（SCREW_GOING_DOWN）
    → 计时 elapsed = now - screw_tick_start
    → 任一条件达成即停止正转：
        ① elapsed_down ≥ screw_limit_down_ms  （软件限位，定时可调）
        ② screw_down_limit硬件限位触发（PE3）
        ③ 步数位置达到 screw_target_fixed
    → 停止后自动进入 STEP2

  STEP2: 反转上升（SCREW_GOING_UP）
    → 计时 elapsed_up = now - screw_tick_start
    → 任一条件达成即停止反转，进入切割准备：
        ① elapsed_up ≥ screw_limit_up_ms    （软件限位，定时可调）
        ② screw_up_limit硬件限位触发（PE2）

保护优先级：硬件限位 > 软件限位定时 > 位置限位
```

**限位时间与丝杆速度换算：**

```
丝杆速度 = screw_speed_hz Hz
步距角（假设1.8°）= 1.8°
全步细分 = 1600（DM860H拨码）

每转步数 = 200（360° / 1.8°）
丝杆导程 = L mm/转（查丝杆规格，如T8丝杆 L=8mm）

丝杆位移速度 = Hz / 每转步数 × 导程 = (screw_speed_hz / 200) × L mm/s

示例（800Hz，导程8mm）：
  位移速度 = (800 / 200) × 8 = 32 mm/s
  正转限位10s → 最大下行距离 = 32 × 10 = 320mm
  反转限位12s → 最大上行距离 = 32 × 12 = 384mm
```

### 4.7 校验和计算（C语言）

```c
uint8_t Protocol_CalcChecksum(uint8_t cmd, uint8_t len, uint8_t* data)
{
    uint8_t cs = cmd ^ len;
    for (uint8_t i = 0; i < len; i++) {
        cs ^= data[i];
    }
    return cs;
}
```

### 4.7 校验和计算（Python）

```python
def checksum(cmd: int, length: int, data: bytes) -> int:
    cs = cmd ^ length
    for b in data:
        cs ^= b
    return cs
```

## 七、ACK响应速查

每条命令执行后 STM32 回传 ACK：

```
帧格式: [AA] [命令+0x80] [02] [状态码] [设备ID/阶段] [校验]
```

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 0x00 | OK | 执行成功 |
| 0x01 | PARAM_ERR | 参数错误 |
| 0x02 | UNSUPPORT | 不支持该命令 |
| 0x03 | BUSY | 设备忙（流程进行中）|
| 0x04 | IDLE | 设备已停止 |

**示例：**

```
TX: AA 30 00 9A              ← 开始工艺
RX: AA B0 02 00 01 B9       ← ACK OK，阶段=01(进料)

TX: AA 31 00 9B             ← 停止工艺
RX: AA B1 02 00 00 9A       ← ACK OK，阶段=00(IDLE)

TX: AA 34 01 01 34          ← 设置清洗模式=仅喷气
RX: AA B4 02 00 01 B7       ← ACK OK，模式=01

TX: AA 35 01 02 34          ← 设置打包模式=中间
RX: AA B5 02 00 02 B7       ← ACK OK，模式=02
```

## 八、状态帧查询（CMD_GET_STATUS）

```
发送: AA 37 00 A7
收到: AA B7 08 [phase][clean][pack][clean_mode][pack_mode][sys][0][0] CS
```

| 字节 | 内容 | 示例值 |
|------|------|--------|
| 0 | 主流程阶段 | `0x01`=进料 |
| 1 | 清洗子状态 | `0x01`=进入清洗 |
| 2 | 打包子状态 | `0x02`=等待葱头 |
| 3 | 清洗模式 | `0x03`=喷气+水泵 |
| 4 | 打包模式 | `0x01`=全部 |
| 5 | 系统状态 | `0x02`=运行中 |
| 6~7 | 保留 | `0x00` |
