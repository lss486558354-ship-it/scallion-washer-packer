## 一、工艺总览

### 1.1 设备组成

```
┌─────────────────────────────────────────────────────────┐
│                    大葱切割-清洗-打包一体机                 │
│                                                         │
│  同步带 ──→ 切割区 ──→ 清洗区 ──→ 传送带 ──→ 打包区     │
│  (步进1)   (步进2丝杆)   (喷气+水)  (DCM1)    (步进3)  │
│           + DCM2刀片    + 舵机SV1    + 称重    + 舵机SV2 │
│                                                         │
│  传感器：同步带光电 | 超声波测距 | 称重传感器 | 打包测距   │
└─────────────────────────────────────────────────────────┘
```

## 二、起保停电路（继电保护）

### 2.1 电气原理

```
                        ┌─────────────────────────────────┐
                        │           起保停回路              │
                        │                                 │
    电源+24V ──┬── [SB_ON 启动按钮 常开] ──┐             │
               │                             ↓             │
    电源+24V ──┼── [SB_OFF 停止按钮 常闭] ─┤── [K 线圈] ──┐  │
               │                             ↑             │  │  │
               │              [K 辅助常开 自锁] ──────────────┘  │
               │                             ↑                    │
               │           [K_S 常开 继电器] ────────────────────┘
               │             (STM32控制)
               │
               └── [急停按钮 常闭] ───→ 接触器K 线圈
                                  ↑ 硬件优先
                              PE6 ──→ STM32 检测

启动流程：
  1. 上电 → 升压板输出DC
  2. 按SB_ON → K线圈得电 → K主触点闭合 → 自锁
  3. 程序启动后 → PA15输出高 → K_S闭合 → 双重保证

停止流程（正常）：
  按SB_OFF → K线圈断电 → 自锁解除 → 断电

急停流程：
  按急停按钮 → K线圈硬件立即断电 → 主电路断开
  同时 → PE6检测到变化 → Process_EmergencyStop() → PA15低 → K_S断
```

## 三、清洗流程详解

### 3.1 子状态机

```
                        ┌───────────┐
                        │  CLEAN_   │ 进入清洗，初始化
                        │  ENTER    │ 清零计时器
                        └────┬──────┘
                             │ 立即
                             ↓
           ┌──────────────────────────────────────────────┐
           │  g_clean_mode 判断                           │
           │                                              │
           │  模式=CLEAN_MODE_AIR_ONLY ──→ CLEAN_SPRAY_ON │
           │  模式=CLEAN_MODE_WATER_ONLY ─→ CLEAN_WATER_ON │
           │  模式=CLEAN_MODE_BOTH ─────→ CLEAN_SPRAY_ON  │
           └──────────────────────────────────────────────┘
                             │
                             ↓
           ┌──────────────────────────────────────────────┐
           │          CLEAN_SPRAY_ON（喷气开启）           │
           │  → 空压机继电器闭合（K_AIR=1）               │
           │  → 舵机SV1摆动到喷气角度                     │
           │  → 延时 clean_air_time_ms                    │
           └────────────┬───────────────────────────────┘
                        │ 喷气时间到
                        ├─→ 模式=仅喷气 ──→ CLEAN_DONE
                        │
                        ↓
           ┌──────────────────────────────────────────────┐
           │          CLEAN_WATER_ON（水泵开启）           │
           │  → 水泵继电器闭合（K_PUMP=1）               │
           │  → 延时 clean_water_time_ms                   │
           └────────────┬─────────────────────────────────┘
                        │ 水泵时间到
                        ↓
                   ┌──────────┐
                   │ CLEAN_   │ 关闭所有设备，舵机复位
                   │  DONE    │ 返回主状态机
                   └──────────┘
```

### 3.2 清洗模式说明

| 模式 | g_clean_mode | 执行流程 | 适用场景 |
|------|-------------|---------|---------|
| 仅喷气 | 0x01 | 喷气开→延时→关闭 | 轻度清洗/去黄叶 |
| 仅水泵 | 0x02 | 等待喷气完成→水泵开→延时→关闭 | 无需喷气 |
| 喷气+水泵 | 0x03 | 喷气→水泵→关闭（默认） | 标准清洗流程 |

> **运行时可切换**：在任何时候发送 `CMD_SET_CLEAN_MODE` 命令可切换清洗模式。

## 四、打包流程详解

### 4.1 测距原理（单传感器）

传感器固定安装在打包工位上方，对着传送带方向：

```
传感器安装位置：
  ┌─────────┐
  │ HC-SR04 │  ← 向下发射超声波
  └────┬────┘
       │ 测距方向
       ↓
  ┌─────────────────────────────────────────────────┐
  │           传送带（大葱运动方向 ←←←）              │
  │                                                  │
  │  空闲时：无大葱 → 超声波→空气 → 距离固定（较大值） │
  │  葱头到：大葱进入 → 距离缩短（变小）              │
  │  葱身在：大葱覆盖 → 距离持续较短                  │
  │  葱尾离：大葱离开 → 距离恢复（变大）              │
  └─────────────────────────────────────────────────┘

判断逻辑：
  距离 < pack_dist_head → 葱头/葱身在下方
  距离 > pack_dist_tail → 葱尾离开/空闲

阈值通过串口可调：
  pack_dist_head（默认55mm）→ 葱头阈值
  pack_dist_tail（默认65mm）→ 葱尾阈值
```

### 4.2 子状态机（三种模式）

#### 模式1：全部打包（PACK_MODE_FULL = 0x01）

```
PACK_ENTER ──→ PACK_WAIT_ONION ──→ PACK_WRAP_ONION ──→ PACK_WAIT_TAIL ──→ PACK_CUT_FILM ──→ PACK_CUT_WAIT ──→ PACK_DONE
                   ↑
              距离<阈值
```

#### 模式2：打包中间（PACK_MODE_MIDDLE = 0x02）

```
PACK_ENTER ──→ PACK_WAIT_ONION ──→ [葱头触发] ──→ PACK_WRAP_ONION
                                                      │
                                                 延时mdidle_time后
                                                      │
                                                      ↓
                                            PACK_WAIT_TAIL ──→ PACK_CUT_FILM ──→ PACK_CUT_WAIT ──→ PACK_DONE
```

#### 模式3：打包头尾（PACK_MODE_HEAD_TAIL = 0x03）

```
PACK_ENTER ──→ PACK_WAIT_ONION ──→ PACK_WRAP_ONION ──→ PACK_CUT_FILM
                   ↑                                        │
                   │                                   切断后
                   │                                    ↓
                   │                              PACK_CUT_WAIT
                   │                                    │
                   │                         延时tail_interval后
                   │                                    ↓
                   └────── PACK_WAIT_TAIL ←────────────┘
```

### 4.3 状态说明

| 状态 | 说明 | 关键动作 |
|------|------|---------|
| PACK_IDLE | 空闲 | 等待进入 |
| PACK_ENTER | 初始化 | 清零标志位，舵机复位 |
| PACK_WAIT_ONION | 等待葱头 | 监控测距，距离<阈值则触发 |
| PACK_WRAP_ONION | 卷膜中 | Motor3正转，卷膜电机运行 |
| PACK_WAIT_TAIL | 等待葱尾 | 监控测距，距离>阈值则切断 |
| PACK_CUT_FILM | 切断膜 | Servo2转到切断角度 |
| PACK_CUT_WAIT | 切断后等待 | 延时后舵机复位 |
| PACK_DONE | 完成 | 停止电机，复位舵机 |

### 4.4 标志位说明

| 标志位 | 说明 |
|--------|------|
| `pack_head_seen` | 葱头已检测到/葱身阶段（卷膜进行中）|
| `pack_tail_seen` | 葱尾已检测到（等待切断）|
| `pack_wrap_done` | 本次卷膜完成 |
| `pack_cut_done` | 切断完成 |

## 五、丝杆与切割联动逻辑

### 5.1 高度调节（软件限位 + 硬件限位）

```
丝杆调整流程（PROCESS_SCREW_ADJUST）：

  ┌────────────────────────────────────────────────────────────┐
  │ STEP1: SCREW_GOING_DOWN  正转下降                         │
  │                                                        │
  │   Motor2.Run(screw_speed, DIR_FORWARD)                  │
  │   计时 elapsed = now - screw_tick_start                  │
  │                                                        │
  │   任一条件满足即停止正转，进入STEP2：                    │
  │   ① elapsed_down ≥ screw_limit_down_ms  ← 软件限位（可调）│
  │   ② screw_down_limit硬件限位触发    ← 硬件限位（保险）   │
  │   ③ 步数达到screw_target_fixed                          │
  │                                                        │
  │   → Device.Motor2.Stop()                               │
  │   → Device.UART1.SendString("[SCREW] Down sw-limit")   │
  │   → 进入STEP2                                          │
  └───────────────────────┬────────────────────────────────┘
                          │ 自动切换
                          ↓
  ┌────────────────────────────────────────────────────────────┐
  │ STEP2: SCREW_GOING_UP  反转上升（回原点）                   │
  │                                                        │
  │   Motor2.Run(screw_speed, DIR_BACKWARD)                  │
  │   计时 elapsed_up = now - screw_tick_start                │
  │                                                        │
  │   任一条件满足即停止反转，进入切割准备：                   │
  │   ① elapsed_up ≥ screw_limit_up_ms   ← 软件限位（可调）   │
  │   ② screw_up_limit硬件限位触发     ← 硬件限位（保险）     │
  │                                                        │
  │   → Device.Motor2.Stop()                                 │
  │   → Device.DCMotor2.Run(cutter_speed, DIR_FORWARD)      │
  │   → 进入 PROCESS_CUTTING_READY                           │
  └────────────────────────────────────────────────────────────┘

软件限位参数（通过串口可调）：
  0x16 screw_limit_down_ms  正转下降最大时间（默认10000ms = 10s）
  0x17 screw_limit_up_ms    反转上升最大时间（默认12000ms = 12s）

保护优先级：硬件限位 > 软件限位定时 > 位置限位
  → 硬件限位任何时候都优先响应（即使软件卡死）
  → 软件限位作为第一道保护，防止撞机
  → 位置限位作为辅助校准

调试方法：
  1. 记录实际运动时间（串口日志查看 elapsed 值）
  2. 将实测时间 × 1.2 作为限位参数
  3. 逐步减小限位时间，直到刚好不撞机
  4. 硬件限位开关保留，防止软件失效时兜底
```

### 5.2 切割联动（深度绑定）

```
切割触发条件：
  1. 丝杆到位（PROCESS_SCREW_ADJUST 完成）
  2. 同步带将大葱送到切割位（PROCESS_CUTTING_READY 延时到达）
  3. 切割电机 DCM2 在 PROCESS_CUTTING_READY 时已启动

切割期间：
  同步带步进1持续正转（切割同时输送）
  切割电机 DCM2 持续运转
  切割时间到达后：
    → DCM2停止（减少危险时间）
    → 步进2反转回原点
    → 进入清洗流程
```

## 六、流程控制命令

### 6.1 串口命令列表

| 命令 | 帧格式 | 说明 |
|------|--------|------|
| `CMD_PROCESS_START` (0x30) | `AA 30 00 CS` | 开始工艺流程 |
| `CMD_PROCESS_STOP` (0x31) | `AA 31 00 CS` | 停止工艺流程 |
| `CMD_PROCESS_PAUSE` (0x32) | `AA 32 00 CS` | 暂停工艺流程 |
| `CMD_PROCESS_RESUME` (0x33) | `AA 33 00 CS` | 恢复工艺流程 |
| `CMD_SET_CLEAN_MODE` (0x34) | `AA 34 01 [模式] CS` | 设置清洗模式 |
| `CMD_SET_PACK_MODE` (0x35) | `AA 35 01 [模式] CS` | 设置打包模式 |
| `CMD_SET_PARAM` (0x36) | `AA 36 03 [ID][值H][值L] CS` | 设置工艺参数 |
| `CMD_GET_STATUS` (0x37) | `AA 37 00 CS` | 查询工艺状态 |

### 6.2 状态回传（CMD_GET_STATUS响应）

```
AA 37 08 [phase][clean_state][pack_state][clean_mode][pack_mode][sys_status][0][0] CS

字节0: 主流程阶段
  0x00=IDLE  0x01=进料  0x02=测高  0x03=调高
  0x04=切割准备 0x05=切割 0x06=清洗 0x07=传送
  0x08=称重  0x09=打包  0x0A=完成
字节1: 清洗子状态
字节2: 打包子状态
字节3: 清洗模式 (0x01=仅喷气 0x02=仅水泵 0x03=全部)
字节4: 打包模式 (0x01=全部 0x02=中间 0x03=头尾)
字节5: 系统状态
字节6~7: 保留
```
