# 飞特总线舵机 — 单片机通讯方法说明

本文说明如何在单片机侧按飞特 **串行总线舵机协议**（与官网《舵机 SCS 通信协议》《磁编码协议手册》一致）组帧、校验并发送。本工程已实现参考代码：`User/servo/bsp_servo_serial.c`（UART4 发送）。

---

## 1. 物理层与串口参数

| 项目 | 说明 |
|------|------|
| 电平 | **SCS 常见**：TTL 半双工单线或 TX/RX 分开；**SM40BL 等 SMS**：RS485 差分，需 MAX3485 等收发器 |
| 数据位 | 8 |
| 校验 | 无 |
| 停止位 | 1 |
| 波特率 | 以舵机内配置为准（常见115200、1000000）；MCU 波特率必须与之一致 |

本工程接线见《外设接线表_硬件PWM步进优化版.md》：**UART4 PC10(TX) / PC11(RX)** 接总线舵机侧（若舵机为单线半双工，需按飞特手册加二极管/电阻网络）。

---

## 2. 数据包格式（指令帧）

字节流顺序：

```text
0xFF 0xFF | ID | LEN | INST | P0 … Pn | CHK
```

| 字段 | 含义 |
|------|------|
| `0xFF 0xFF` | 固定帧头 |
| `ID` | 舵机地址0～253；`0xFE` 为广播（多数指令无应答） |
| `LEN` | **从 INST 到 CHK（含）** 的总字节数，即 `1 + (参数个数) + 1` |
| `INST` | 指令码，如读 `0x02`、写 `0x03`、PING `0x01` |
| `P0…` | 参数；**写寄存器**时第一个参数为**起始地址**，其后为写入数据 |
| `CHK` | 校验和，见下节 |

**写寄存器（INST = 0x03）** 时：参数区为 `[起始地址][数据0][数据1]…`，可多字节连续写。

---

## 3. 校验和 CHK

对 **ID、LEN、INST 及所有参数字节** 做 8 位无进位累加，再按位取反，取低 8 位：

```c
uint8_t scs_checksum(uint8_t id, uint8_t len, uint8_t inst,
 const uint8_t *params, uint8_t nparam)
{
    uint16_t s = (uint16_t)id + len + inst;
    uint8_t i;
    for (i = 0; i < nparam; i++)
        s += params[i];
    return (uint8_t)(~((uint8_t)s));
}
```

其中 `params` 在写指令中为 **`[起始地址] + 数据字节`**，`nparam = 1 + 数据长度`。

**例**：PING ID=1：`FF FF 01 02 01 FB`  
累加：`0x01+0x02+0x01 = 0x04`，`~0x04 = 0xFB`。

---

## 4. 常用写指令示例（磁编码 / SCS 常用小端）

以下多字节数值均为 **低字节在前**（具体型号以该型号**内存表**为准）。

### 4.1 写目标位置块（首地址 0x2A，连续 6 字节）

顺序：**目标位置 16bit | 目标时间 16bit | 目标速度 16bit**，均小端。

与手册例 4 一致：ID=1，位置 0x0800，时间 0，速度 0x03E8(1000)：

```text
FF FF 01 09 03 2A 00 08 00 00 E8 03 D5
```

### 4.2 写目标速度（轮模式，寄存器 0x2E，有符号 16 位小端）

舵机需先处于**轮模式**（上位机或内存表配置）。`speed` 为 `int16_t`，小端写入。

---

## 5. 单片机发送流程（通用步骤）

1. 配置 UART：波特率、8N1，TX（及半双工时 RX）引脚。  
2. 按上表组缓冲区：`FF FF` + `ID` + `LEN` + `INST` + 参数… + `CHK`。  
3. **LEN** 必须等于 `1 + 参数字节数 + 1`（含 INST 与 CHK）。  
4. 依次 `UART_SendByte` 或 `DMA` 发送；帧间可按手册留短延时（多机总线时更稳）。  
5. 若需读应答：在发送后切换接收（半双工总线需硬件支持），解析返回帧（本工程当前为**仅发不收**的调试路径，读状态需另行开 RX与超时处理）。

## 7. 注意事项

1. **字节序**：磁编码舵机多数字段为小端；个别老型号电位器版本为大端，**务必核对内存表**。  
2. **SM40BL / RS485**：MCU 经 RS485 芯片接总线时，DE/RE 半双工方向脚需在发送前使能发送、发送后切回接收（若需读应答）。  
3. **供电与共地**：逻辑地与大电流舵机电源地按手册连接，避免串口地弹导致误码。  
4. **官方文档**：最新寄存器地址、运行模式、同步读写指令以飞特官网下载的 PDF 为准。  
5. **飞特 URT-1 调试板**：板载 **CH340**，**Micro USB 只接电脑（USB Host）**；TTL 侧接舵机总线（半双工按手册）。**不能把 STM32 的 USB 口去插 URT-1 的 USB口**指望经 CH340 转 TTL：F103 与 CH340 在 USB 上都是 **Device**，无 Host 无法枚举通讯；控舵机请用 **UART4** 发本文数据包。详见《STM32F103_USB与URT1舵机通讯说明.md》§4。

## 8. 排查：USB-TTL 经 URT-1 不能通讯

**现象**：Micro USB 插电脑 → URT-1 → 舵机 **正常**；改用 **独立 USB-TTL 模块**（CH340/CP2102 等）接 URT-1 再舵机 **无反应**。

常见原因与对策如下（按优先级自查）。

### 8.1 误把 TTL 接到 Micro USB 口

- **Micro USB 座**上是 **USB 差分信号（D+/D-）**，**不是** 异步串口的 TX/RX。  
- **USB-TTL 模块**上 **只有** `TX`、`RX`、`GND`、`VCC` 等 **TTL 脚**可以接 UART；**绝不能**把这些脚接到 URT-1的 **USB 数据脚**或指望「像插电脑一样」用杜邦线模拟 USB。  
- **结论**：若你的接法是「USB-TTL 的 TX/RX 接到 URT-1 的 USB 口附近或 USB 座焊盘」→ **必然不通**，请改接法或放弃「TTL 再进 URT-1 USB」这条路。

### 8.2 飞特 URT-1（带底部5 针排针）外接 USB-TTL 的正确接法

常见 **URT-1** 底板底部有 **5 针**：丝印一般为 **TXD、RXD、+5V、GND、DTR**（以你板子为准）；右侧 **SCS** 为 **三芯 G / V1 / S**（单线半双工），左侧 **SMS** 为 **RS485（A/B）**，与 SCS **不能混用**。实物丝印可参考工程内图片：`project/URT-1_board_photo.png`。

**用外接 USB-TTL 走 URT-1 再控 SCS 舵机时**（**不要再插 URT-1 的 Micro USB 到电脑**，避免两路串口同时进板子）。

**TX/RX 线序**：教材上常写「一方 TX 接对方 RX」，但 **URT-1 底排丝印与部分 USB-TTL 模块** 在实际产品上可能出现 **同名字直连即通** 的情况（与板内走线/标注习惯有关）。请按下表 **二选一** 尝试，**以你手上能稳定通讯为准**：

| 接法 | USB-TTL → URT-1 排针 | 说明 |
|------|----------------------|------|
| **A. 同丝印直连（实测可通）** | **TX → TXD**，**RX → RXD** | 本工程使用者已在 **115200** 下用此接法与串口舵机正常通讯。 |
| **B. 经典交叉** | **TX → RXD**，**RX → TXD** | 若 A 不通再试此项（一方发脚接另一方收脚）。 |

**其余引脚**：

| USB-TTL 模块 | URT-1 排针 | 说明 |
|--------------|------------|------|
| **GND** | **GND** | 必须共地 |
| **5V（可选）** | **+5V** | 若转接板逻辑需供电：可由 **USB-TTL 的 5V** 供 **+5V**（注意 USB 口电流能力）；**舵机动力仍接 V1 端子** |
| **DTR** | **DTR** | 若飞特手册或你模块支持半双工方向控制，再按说明接；否则可先 **悬空** 试 |

另须同时满足：

1. **SCS 舵机** 插在 **SCS 口**（**G、V1、S**），不要插在 **SMS（RS485）** 口。  
2. **V1 螺丝端子** 按丝印接 **6～9 V**（SCS 供电），**G** 与舵机地共地；无电则舵机与板上电路可能都不工作。  
3. 板载拨动开关 **3V3 / VCC / 5V**：与外接 USB-TTL **电平一致**（常见 **5V** 档配合 5V TTL 模块）。  
4. **波特率**：与舵机、串口软件、单片机 UART **一致**；**SCS 串口舵机常见为 115200**（以舵机出厂/调参为准）。

若 A、B 仍不通：测 **TXD/RXD** 对地是否有波形、核对 **DTR** 是否必接（查 URT-1 说明书）。

**旧款/简化 URT-1** 若确实 **没有** 这 5 针，则无法「外接 TTL 进板」，只能 **Micro USB 接电脑** 或 **绕过转接板自搭半双工**（见 §8.6）。

### 8.3 其它带 UART 排针的转接板

- **GND必共地**；**TX/RX** 若交叉不通可改 **同丝印直连** 再试（与 8.2 同理）。  
- **波特率**（如 115200）、**电平** 与舵机一致。

### 8.4 供电与共地

- 仅用 USB-TTL 时，若 URT-1 **原靠电脑 USB 供电**，改接后可能 **未供电** 或 **地未与舵机/模块共好**，导致 CH340 或缓冲电路不工作。  
- 确认：**URT-1 / 舵机逻辑地** 与 **USB-TTL GND** 与 **电脑地（通过 USB）** 形成可靠参考地（按实际接线图）。

### 8.5 小结（推荐路径）

| 需求 | 推荐接法 |
|------|----------|
| 用飞特板载 CH340 | **电脑 USB → URT-1 Micro USB → 舵机**（你已验证可行） |
| 用自己 USB-TTL | **电脑 USB → USB-TTL →（任选其一）① 半双工电路 → 舵机**；或 **② USB-TTL ↔ URT-1 底部 TXD/RXD/GND 排针**（见 8.2，**不要**把 TTL 接到 Micro USB 的 D+/D-） |
| 单片机控舵机 | **MCU UART4（等）→ 半双工电路 → 舵机**，见本文前文 |

### 8.6 三芯单线舵机：为何「TX 接 RX、RX 接 TX、GND 共地」仍不通

飞特 **3 芯**舵机一般为：**电源正、GND、一根 Signal（半双工总线）**。接口上 **没有** 独立的「舵机 TX」「舵机 RX」两个脚。

| 误区 | 说明 |
|------|------|
|按「两个串口设备」交叉 TX/RX | 这只适用于 **双方都有 TX/RX 的 UART**（例如两块板子对接）。舵机侧 **只有一根信号线**，不能把 USB-TTL 的 TX、RX **当成两根线分别接舵机两个脚**（除非你的舵机型号确实是 **4 线 UART分开 TX/RX**，以该型号手册为准；SCS 常见 3 芯则不是）。 |
| 省略半双工电路 | USB-TTL 输出的是 **TX、RX 两路 TTL**。接到 **单线总线** 前，必须按飞特文档用 **二极管 + 电阻** 等把发送与接收 **汇合到同一根线**，再接到舵机 Signal。**不能把 TX 与 RX 直接拧在一起接舵机。** |
| 电路抄错 | 二极管方向、阻值、是否需上拉，以官网 **单线调试/转接原理图** 为准；与 URT-1 板载已布好的网络等效，自搭易错。 |

**建议正确理解**：

1. **GND**：USB-TTL **GND**、舵机 **GND**、舵机电源地 **必须共地**。  
2. **电源**：舵机 **VCC 按手册用电池/稳压电源供电**；不要指望 USB-TTL 的 5V 长期带大扭矩电流。  
3. **信号**：USB-TTL 的 **TX、RX 先进入「半双工转换电路」**，电路 **输出一根线** 接舵机 **Signal**（与 URT-1 接到舵机的那一路 **等效**）。  
4. **波特率**：串口助手 / `scs_servo_host` 与舵机内波特率 **完全一致**（常见 115200 或 1000000）。  
5. **电平**：舵机多为 **5V TTL**；纯3.3V USB-TTL 若不通，可换 **5V 兼容**模块或加电平转换再试。

**若暂时只有模块、想先验证「能否发出字节」**（仍建议尽快按官方图搭半双工）：可查阅飞特 PDF 中 **「单线串口」或「调试器」** 章节里的 **限流电阻 + 二极管** 接法；**不要**在未确认的情况下把 TX/RX 短接。

**与 URT-1 对比**：URT-1 能通，说明舵机与协议正常；自搭不通多半是 **半双工电路未按图实现** 或 **仍按双线交叉理解三芯舵机**。可对照 URT-1 原理图（若有）或官网 **SCS 单线接口** 应用笔记逐点核对。
