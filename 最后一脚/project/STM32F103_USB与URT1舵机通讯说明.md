# STM32F103VET6 USB 与飞特 URT-1（CH340）舵机通讯说明

## 1. 芯片 USB 功能是否可用

**可以。** STM32F103xE（含 VET6）属于中等容量以上产品，片内集成 **USB 全速设备（Full-Speed Device）**外设。

| 项目 | 说明 |
|------|------|
| 引脚 | **PA11 = USB_DM**，**PA12 = USB_DP**（固定，**不可重映射**） |
| 电气 | 需按 USB 规范走线、D+/D- 阻抗与 ESD；开发板若已带 **mini/micro USB 座与33Ω 串联电阻**，可直接用 |
| 时钟 | USB 外设需要 **48 MHz**时钟；典型配置为 SYSCLK=72 MHz，PLL 后由硬件 **÷1.5** 得到 48 MHz 供 USB（参考 RM0008） |

固件侧需实现 **USB 设备枚举**（常见为 **CDC 类虚拟串口**），PC 才会出现“虚拟 COM 口”。本工程 **当前未集成** USB 协议栈，USART/UART4 等与 USB 无关。

---

## 2. 与本工程引脚是否冲突

**按《外设接线表_硬件PWM步进优化版》与当前 `bsp_stepper.c`：步进 5/6 已使用 TIM1 FullRemap 的 PE14/PE13，不再使用 PA11/PA12 作步进 PUL。**

文档中已写明：**PA11 为 USB_DM**，若使用 USB，则 **不能把 PA11 当作 TIM1 或其它复用输出**。

请注意：

- 遗留模块 **`bsp_motor_obj.c` / `bsp_servo_obj.h` 注释** 仍可能出现 “PA11/PA12 + TIM1” 的旧描述，与 **现行步进硬件 PWM 方案不一致**；以 **`bsp_stepper.c` + 接线表** 为准。
- **调试文件** `stepper_debug_test.c`、`bsp_stepper_debug.c` 若参与编译，可能再次配置 PA11/PA12；**启用 USB 时不要使用这些把 PA11/PA12 当作 TIM 的调试代码**。

**结论：在正式步进固件路径下，PA11/PA12 可为 USB 让出；启用 USB 前请全局确认无其它驱动占用这两脚。**

## 3. 飞特 URT-1 调试板（板载 CH340）

**URT-1 上带有 CH340**（常见为 CH340G 等，以板子丝印为准）：**Micro USB 只接电脑（USB Host）**，在 PC 上枚举为 **USB 转串口（COM）**。不少型号在板底另有 **5 针排针（TXD、RXD、+5V、GND、DTR）**：可把 **外接 USB-TTL** **共地** 后接 **TX/RX**——实测常见 **USB-TTL 的 TX 接 URT-1 的 TXD、RX 接 RXD（同丝印直连）** 即可通讯；若不通再试 **TX 接 RXD、RX 接 TXD（交叉）**；由板载电路完成 **半双工** 后接到 **SCS 口（G / V1 / S）**。**勿**把 TTL 杜邦线接到 Micro USB 座的 **D+/D-**。波特率等与舵机一致（**SCS 常用 115200**），细节见《飞特总线舵机_单片机通讯方法》第 8.2 节。  
也就是说：**CH340 或外接排针 UART** 负责「主机侧全双工 TTL」；**板载转换** 后才是舵机总线；舵机侧协议与 STM32 用 **UART4** 发帧是同一类信号层（电平、半双工接法需符合手册）。排针接法细节见《飞特总线舵机_单片机通讯方法》第 8 节。

## 4. 专门说明：能否「单片机 USB」插 URT-1 的「Micro USB」经 CH340 再 TTL 到舵机？

**不能按这种方式工作。** 常见误解是：URT-1 上 **USB 口 ↔ CH340 ↔ TTL**，于是想用 **STM32 的 USB（PA11/PA12接到 URT-1 的 USB）** 当作「输入」，让 CH340 在板子上自动变成 TTL 去舵机——这在 **USB 协议与 F103 能力上都不成立**。

| 要点 | 说明 |
|------|------|
| USB 拓扑 | 标准 USB 需要 **主机（Host）** + **设备（Device）**。电脑是 Host；**CH340 在 URT-1 上是 Device**（插电脑才被枚举）。 |
| STM32F103 | 片内 USB 仅为 **全速 Device**，**不是 USB Host / OTG Host**，**不能**扮演主机去「识别」插在另一头的 URT-1。 |
| 两根线互插 | 把 **STM32 开发板的 USB 座** 与 **URT-1 的 Micro USB** 用线连在一起，等于 **Device 对 Device**，**没有 Host 做枚举与调度**，**无法正常通讯**，更不会自动在 CH340 的 TTL 侧输出你给舵机的串口数据。 |
| 正确信号层 | 舵机总线要的是 **UART 时序的 TTL/半双工**；应由 **STM32 的 UART（本工程 UART4）** 接入飞特手册规定的总线电路，**不要**试图用 USB 差分线替代这一路。 |

**若目标仍是「单片机发指令、舵机动作」**：请使用 **`UART4（PC10/PC11）` + 半双工接口** 接舵机总线（与《飞特总线舵机_单片机通讯方法》一致）。**URT-1** 仅在 **电脑** 需要调参/抓包时，由 **PC 的 USB** 接 URT-1；**PC 与 STM32 不要指望通过「STM32 USB 插 URT-1 USB」这条路径串通 CH340。**

若希望 **电脑 ↔ USB ↔ 单片机** 再间接控舵机，应走 **STM32 跑 USB CDC（虚拟串口）**，固件里把收到的字节 **转发到 UART4**，而不是去接 URT-1 的 USB 口。

## 5. “把 STM32 的 USB 接到 CH340 的 TTL 脚” 的误区与可行接法

### 5.1 不能这样做的原因

**URT-1 上 CH340** 与电脑相连时，其作用是：

- **USB 口**：与 **PC** 相连，在电脑上枚举为 **USB 转串口（COM）**；
- **TTL 侧**：**TX/RX** 为 **异步串口电平**，与舵机总线协议层一致，但 **不是 USB 差分信号**。

STM32 的 **PA11/PA12** 是 **USB D-/D+**，**绝不能**与 CH340 的 **TX、RX** 或 **3.3V TTL** 引脚直接相连，否则电气与协议均错误，可能损坏器件。

### 5.2 推荐架构（二选一或组合）

**方案 A — PC 经 URT-1 调舵机（与单片机并联总线时要谨慎）**

1. **PC —USB→ URT-1（CH340）—TTL→** 飞特总线（半双工电路按手册）→舵机。  
2. 单片机若也要控制同一总线：用 **UART4（PC10/PC11）** 经 **同一半双工总线规范** 接入；注意 **同一时刻避免两台主机同时强驱**，必要时加缓冲/开关或分时上电调试。

**方案 B — PC 经 STM32 的 USB 虚拟串口，再由 MCU 转发到舵机**

1. **PC —USB→ STM32（PA11/PA12，CDC）**；固件从 USB 收字节，组帧或透传，由 **UART4** 发往舵机总线。  
2. 此路径 **不经过 CH340**；CH340 仅在“PC 没有直连 STM32 USB、只有 USB-串口小板”时作为 **另一套 PC 侧工具**使用。

**方案 C — 仅调试：PC 只连 URT-1，STM32 不参与**

适合纯飞特上位机/串口助手经 URT-1 控制舵机；与 MCU 固件中的 `bsp_servo_serial` 无关。

## 6. 若要在本工程启用 USB CDC（概要步骤）

以下为方向性清单，具体需引入 ST 官方或社区 **USB Device 库**（与当前 Keil 工程风格对齐）：

1. **系统时钟**：保证 USB 48 MHz 时钟路径正确（72 MHz 典型配置下经硬件除以 1.5 得到 48 MHz）。  
2. **GPIO**：PA11/PA12 配置为 **USB 复用**（参考手册 AFIO）。  
3. **中断**：`USB_LP_CAN1_RX0_IRQn` / `USB_HP_CAN1_TX_IRQn` 等按所用库填写向量表。  
4. **协议栈**：实现 **Composite 或单一 CDC**，枚举为虚拟 COM；接收环形缓冲与 **`ServoBus_WriteReg` / UART4_SendData`** 之间做协议解析或透传。  
5. **与 UART4 舵机**：逻辑上等同“多一个来自 PC 的命令入口”，注意缓冲与实时性。

工作量大于“改几行 UART”，建议单独分支或参考 ST **STM32_USB-FS-Device_Driver** / **VCP** 例程（同芯片系列）。

## 7. 小结

| 问题 | 结论 |
|------|------|
| VET6 有无 USB？ | 有，PA11/PA12。 |
| 当前步进方案占不占 PA11/PA12？ | **主路径不占**；勿与旧 motor 调试、TIM1 无重映射方案混用。 |
| 能否把 STM32 USB 线接到 CH340 的 UART？ | **不能。** |
| 能否 STM32 的 USB 插 URT-1 的 USB，借 CH340 出 TTL 控舵机？ | **不能。** 双方都是 USB **Device**，无 **Host**；F103 也无 USB Host。 |
| URT-1 怎么用？ | **仅 PC 的 USB** 接 URT-1；CH340 的 TTL 经半双工电路接舵机总线。MCU 控舵机用 **UART4** 接同一总线（注意勿双线同时强驱）。 |
| 想用 USB 连 PC 再控舵机？ | 在 MCU 上跑 **USB CDC**，再 **转发到 UART4**；或继续用 **USART2（Jetson）/调试串口** 等现有通道。 |
