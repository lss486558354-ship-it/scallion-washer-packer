## 三、打包工艺详解（核心改进区）

### 3.1 旧方案问题

**问题描述**：打包环节中膜杆第二节（ID2/ID7）不动作，导致打包不完整。

**根因分析**（基于固件 `apply_changes.py` 注释）：

1. 旧分区D的 `g_pack_test_state` 状态机使用 `PACK_TEST_IMPELLER_ON` 触发打包，`g_pack_test_seg_ms` 仅记录当前阶段时间，缺乏全流程统一计时
2. 膜杆分两节运行的控制逻辑与打包阶段总时间 `PACK_TEST_IMPELLER_DURATION_MS` 的判断存在时序冲突
3. 分区E（超声波计数）和分区H（旧打包状态机）冗余，状态流转不清晰

### 3.2 新方案架构

**核心改进**：新建【分区F】切割+打包联合状态机，替代原分区D/E/H。

```
【分区 F】切割+打包联合状态机
触发：USART3 收到 0x9A 命令（或调试串口命令）

  IDLE ──(0x9A)──→ BELT_RUN ──(超声<阈值)──→ CUT_RUN
    ↑                   ↑                        │
    │                  IR计数<3              IR计数≥3
    │                    │                    ↓
    └──────PACK_STOP←──PACK_ROD_ON_2←───PACK_IMPELLER
                                       │B7低   │B7高
                                  PACK_M4     PACK_ROD_ON_1
```

### 3.3 各状态详细说明

| 状态 | 说明 | 关键动作 |
|------|------|---------|
| `PROC_NEW_IDLE` | 等待0x9A启动命令 | 所有设备停止 |
| `PROC_NEW_BELT_RUN` | 同步带CW运转+红外计数 | 每60ms轮询超声波，<50mm触发切割 |
| `PROC_NEW_CUT_RUN` | 切割电机运行1.5s | PC9输出高驱动切割，8s内判断IR计数≥3则进打包流 |
| `PROC_NEW_PACK_IMPELLER` | M3+叶轮3/8启动 | 扭矩使能→设置速度→B7低时触发M4 |
| `PROC_NEW_PACK_M4` | M4运行中（M3+叶轮继续） | 距离 `PACK_TEST_ROD_ON_MS - 1000ms` 触发动膜杆 |
| `PROC_NEW_PACK_ROD_ON_1` | 膜杆2/7第一节 | 停止叶轮→判断方向→轮模式运行，固定时长 `PACK_TEST_ROD_ON_MS + PACK_TEST_ROD_LEG_MS` |
| `PROC_NEW_PACK_ROD_OFF` | M3+M4+叶轮恢复 | `PACK_TEST_M3_RECOVER_MS` 后恢复M3/M4/叶轮，等B7从低变高 |
| `PROC_NEW_PACK_ROD_ON_2` | 膜杆2/7第二节 | B7高时开始，计时1950ms后停止膜杆 |
| `PROC_NEW_PACK_STOP` | 全部停止，返回IDLE | 停止所有设备，清零计数 |

### 3.4 打包阶段总时间机制

**新增变量**：`g_pack_test_total_ms` —— 打包阶段绝对累计时间

- 首次进入 `PACK_IMPELLER` 时记录 `s_pack_start_tick`
- 每次进入打包相关状态时，`g_pack_test_total_ms = now_tick - s_pack_start_tick`
- 当 `g_pack_test_total_ms >= PACK_TEST_IMPELLER_DURATION_MS` 时，安全停止所有设备

### 3.5 膜杆分节控制逻辑

```
膜杆第一节 (PACK_ROD_ON_1)：
  - 停止 M3/M4/叶轮
  - 读取 ID2/ID7 当前位置，判断方向（抽出 vs 推入）
  - 以轮模式运行一段时间后停止

膜杆第二节 (PACK_ROD_ON_2)：
  - 等待 B7 信号从低变高（膜已到位）
  - 再次启动膜杆运行 1950ms
  - 完成后进入 PACK_STOP
```

## 四、通信协议

### 4.1 物理串口映射

| 串口 | 用途 | 协议 |
|------|------|------|
| USART1 | 调试打印/HX711 | 文本 + ASCII命令 |
| USART2 | 香橙派通信 | 0xAA帧协议 |
| USART3 | 淘晶驰串口屏 | 0xAA帧协议（屏事件驱动） |
| UART4 | 飞特总线舵机 | `FF FF ID LEN INST ... CHK` |

### 4.2 核心下行命令（STM32 ← 上位机）

| CMD | 名称 | 数据格式 | 说明 |
|-----|------|---------|------|
| 0x30 | CMD_PROCESS_START | 无 | 开始工艺流程 |
| 0x31 | CMD_PROCESS_STOP | 无 | 停止工艺流程 |
| 0x35 | CMD_SET_PACK_MODE | [模式] | 设置打包模式 |
| 0x36 | CMD_SET_PARAM | [ID][值H][值L] | 设置工艺参数 |
| 0x37 | CMD_GET_STATUS | 无 | 查询工艺状态（回8字节） |
| 0x38 | CMD_ULTRASONIC_TEST | 无 | 测试超声波（回3字节） |

### 4.3 总线舵机速度编码

**STS3020舵机速度寄存器编码**（`bsp_servo_serial.c`）：

- 正速度 = 原值（0~1000）
- 负速度 = 1024 + |speed|（有符号转无符号）
- 大端序（BigEndian）写入寄存器 0x2E
- 轮模式初始化6步：关扭矩→解锁→角限位清零→写模式寄存器→锁定→开扭矩
