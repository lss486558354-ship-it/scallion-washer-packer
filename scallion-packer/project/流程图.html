4分区状态详情

🤚 分区A：夹入切割 ZONE_A
  • CLAMP_IDLE - 空闲
  • CLAMP_GRIP - 夹入动作
  • CLAMP_US_SENSE - 超声波检测
  • CLAMP_SCREW_ADJUST - 丝杆调整
  • CLAMP_SCREW_HOLD - 丝杆保持
  • CLAMP_CUT - 切割动作
  • CLAMP_DONE - 完成
💧 分区B：清洗 ZONE_B
  • CLEAN_IDLE - 空闲
  • CLEAN_ENTER - 进入
  • CLEAN_SPRAY_ON - 喷气开启
  • CLEAN_SPRAY_OFF - 喷气关闭
  • CLEAN_WATER_ON - 水泵开启
  • CLEAN_WATER_OFF - 水泵关闭
  • CLEAN_DONE - 完成
⏳ 分区C：等待 ZONE_C
  • WAIT_IDLE - 空闲(清零计数)
  • WAIT_DELAY - 定时等待
  • WAIT_READY - 等待完成
📋 判断条件：onion_count ≥ target_count ?
📦 分区D：打包 ZONE_D
  • PACK_ZONE_IDLE
  • PACK_ZONE_ALIGN_REV - 对齐反转
  • PACK_ZONE_FEED_FWD - 正转夹入
  • PACK_ZONE_WAIT_PACK - 等葱到达
  • PACK_ZONE_WRAP_START - 缠膜开始
  • PACK_ZONE_ROD_PULL - 膜杆抽离
  • PACK_ZONE_WRAP_TAIL - 等葱尾离开
  • PACK_ZONE_WRAP_COMPLETE
  • PACK_ZONE_EJECT - 抛出
  • PACK_ZONE_DONE

伪多线程实现机制

⚡ SysTick 1ms 中断

硬件定时器每1ms触发一次，执行时间基准更新

void SysTick_Handler(void) { g_tick_ms++; BSP_Tick_AppHook1ms(); }

🔄 pending 机制

ISR设置标志，主循环执行任务，避免ISR阻塞

// ISR: 每10ms置位 if (++div >= 10) { div = 0; s_pending++; } // 主循环: 执行任务 while (s_pending) { s_pending--; Process_Main_SM_Run(); }

⏱️ 计数器分时

每个分区用独立计数器管理时间，实现分时复用

// 电机3/4自检 if (s_m34_boot_cnt == 0) { Stepper_RunAtSpeed(...); } if (++s_m34_boot_cnt >= seg_ms) { s_m34_boot_flag++; }

🔀 状态机分派

通过g_proc_state决定执行哪个分支，实现逻辑隔离

switch (g_proc_state) { case PROC_FLOW1_SYNC: ... case PROC_FLOW2_SCREW_CCW: ... case PROC_FLOW3_CONVEY: ... }
