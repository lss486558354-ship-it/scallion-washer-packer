## 一、硬件接线

### 1.1 串口连接总览

STM32F103VE 共有 3 个串口：

| 串口 | TX 引脚 | RX 引脚 | 用途 |
|------|---------|---------|------|
| **USART1** | PA9 | PA10 | 调试串口 / **语音模块** |
| **USART2** | PA2 | PA3 | Jetson Orin Nano 通信 |
| **USART3** | PB10 | PB11 | 预留扩展 |

> **语音模块建议连接 USART1（PA9/PA10）**，与 Jetson 的 USART2 互不干扰。

### 1.2 语音模块接线（USART1）

```
语音模块                       STM32F103VE
┌─────────────┐              ┌─────────────┐
│  TX         │◄───线──────►│  PA10 (RX)  │
│  RX         │◄───线──────►│  PA9  (TX)  │
│  GND        │◄───线──────►│  GND        │
└─────────────┘              └─────────────┘
```

### 1.3 Jetson Orin NX / Orin Nano 接线（USART2）

```
Jetson 40Pin                        STM32F103VE
┌──────────────────┐              ┌─────────────┐
│ Pin 8  (UART1_TX)│◄───线───────►│ PA3 (RX)    │
│ Pin 10 (UART1_RX)│◄───线───────►│ PA2 (TX)    │
│ Pin 6  (GND)     │◄───线───────►│ GND         │
└──────────────────┘              └─────────────┘

## 二、协议帧格式

所有通信均使用以下帧格式，**帧头 0xAA** 起始：

```
┌────────┬────────┬────────┬─────────────┬────────┐
│ 帧头   │ 命令   │ 长度   │ 数据         │ 校验   │
│ 1字节   │ 1字节  │ 1字节  │ N字节        │ 1字节  │
├────────┼────────┼────────┼─────────────┼────────┤
│  0xAA  │ CMD    │  len   │ data[0..N-1] │  CS    │
└────────┴────────┴────────┴─────────────┴────────┘
```

**校验和计算（异或校验）：**

```
CS = CMD ^ len ^ data[0] ^ data[1] ^ ... ^ data[len-1]
```

**校验和计算示例（步进1启动，速度2000Hz）:**

```
命令  = 0x22
长度  = 3
数据  = [0x01, 0x07, 0xD0]   ← [电机ID, 速度高8位, 速度低8位]

CS = 0x22 ^ 0x03 ^ 0x01 ^ 0x07 ^ 0xD0
   = 0xF7

完整帧: AA 22 03 01 07 D0 F7
```

## 四、指令详解

### 4.1 直流电机启动 — CMD_DCM_START (0x20)

**数据格式：** `[电机ID] [占空比] [方向(可选)]`

| 字段 | 字节数 | 说明 |
|------|--------|------|
| 电机ID | 1 | 0x11=DCM1, 0x12=DCM2, 0x13=DCM3, 0x1F=全部 |
| 占空比 | 1 | 0~100（百分比） |
| 方向 | 1 | 0x00=正转, 0x01=反转（可选） |

> **重要：** 此命令为绝对值设置，执行时会**自动清零**该电机的偏移量。

**示例：**
```
DCM1（传送带）正转80%：
  AA 20 02 11 50 43
  校验 = 0x20 ^ 0x02 ^ 0x11 ^ 0x50 = 0x43

DCM2（刀片1）反转60%：
  AA 20 03 12 3C 01 2F
```

### 4.2 步进电机启动 — CMD_STEPPER_START (0x22)

**数据格式：** `[电机ID] [速度高8位] [速度低8位]`

速度（Hz）转换为高/低字节：
```
2000 Hz → 高 = 2000 / 256 = 7 (0x07)  低 = 2000 % 256 = 208 (0xD0)
实际帧: [ID] [0x07] [0xD0]
```

**示例：**
```
步进1（切割）启动2000Hz：
  AA 22 03 01 07 D0 F7

步进2（传送带）启动1500Hz：
  AA 22 03 02 05 DC F3

步进3（打包）启动3000Hz：
  AA 22 03 03 0B B8 F1
```

### 4.3 通用速度+方向 — CMD_SET_SPEED_DIR (0x24)

**数据格式：** `[电机ID] [类型] [方向] [值高8位] [值低8位]`

这是一条最重要的综合指令，可同时设置速度和方向，适用于需要精确控制的场景。

**示例：**
```
步进1正转2000Hz：
  AA 24 05 01 01 00 07 D0 57

步进1反转2000Hz：
  AA 24 05 01 01 01 07 D0 56

DCM1反转80%：
  AA 24 05 11 02 01 00 50 77

DCM2正转60%：
  AA 24 05 12 02 00 00 3C 2F
```

### 4.4 方向设置 — CMD_STEPPER_DIR (0x25) / CMD_DCM_DIR (0x26)

**步进电机方向 (0x25)：**
```
AA 25 02 01 00 xx   ← 步进1正转
AA 25 02 01 01 xx   ← 步进1反转
```

**直流电机方向 (0x26)：**
```
AA 26 02 11 00 xx   ← DCM1正转
AA 26 02 11 01 xx   ← DCM1反转
```

> 注意：直流电机方向设置**不影响当前运行状态**，只改变下次 `DCM_START` 时的方向。

## 五、ACK响应机制

每条命令执行后，STM32 会自动回传一个 **ACK 帧**，通过**原路返回**（哪个串口收到命令就从哪个串口返回）。

### 5.1 ACK帧格式

```
[AA] [原命令+0x80] [02] [状态码] [设备ID] [校验]
```

**示例：**
```
发送: AA 20 02 12 3C 2E        ← DCM2启动60%
收到: AA A0 02 00 12 92        ← ACK OK，设备ID=0x12
      命令=A0=0x20+0x80，状态=00(OK)，设备=12
```

### 5.2 状态码定义

| 状态码 | 含义 | 说明 |
|--------|------|------|
| `0x00` | OK | 执行成功 |
| `0x01` | PARAM_ERR | 参数错误（ID或长度不对） |
| `0x02` | UNSUPPORT | 不支持该命令 |
| `0x03` | BUSY | 设备忙 |
| `0x04` | IDLE | 设备已停止 |

### 5.3 ACK解析示例

```
TX: AA 20 02 12 3C 2E    ← DCM2启动60%
RX: AA A0 02 00 12 92    ← ACK OK，设备0x12执行成功

TX: AA 21 01 12 33        ← DCM2停止
RX: AA A1 02 00 12 93    ← ACK OK，设备0x12已停止

TX: AA 2F 00 2F           ← 全部急停
RX: AA AF 02 00 00 9D    ← ACK OK，全部设备急停完成
```

## 六、偏移量控制（语音加速/减速）

### 6.1 设计原理

通过**偏移量全局变量**实现"加速/减速"功能，不干扰绝对值设置：

```
实际值 = 设置值 + 偏移量
```

| 变量 | 类型 | 范围 | 适用 |
|------|------|------|------|
| `g_dcm_offset[0~2]` | int8_t | -100 ~ +100 | 直流电机占空比 |
| `g_stepper_offset[0~2]` | int16_t | -20000 ~ +20000 | 步进电机速度 |

### 6.2 偏移量命令

**调整偏移量 — CMD_OFFSET_ADJUST (0x27)**

```
格式: [电机ID] [类型] [偏移高8位] [偏移低8位]
类型: 0x01=步进电机  0x02=直流电机
偏移: 有符号16位整数（struct.pack('<h')自动处理符号）
```

**示例：**
```
DCM2（刀片1）占空比 +10：
  AA 27 04 12 02 00 0A B9
  校验 = 0x27 ^ 0x04 ^ 0x12 ^ 0x02 ^ 0x00 ^ 0x0A = 0xB9

DCM1占空比 -5：
  AA 27 04 11 02 FF FB 31
  偏移量 = -5 (0xFFFB as int16)

步进1速度 +500：
  AA 27 04 01 01 01 F4 57
  偏移量 = +500 (0x01F4)

步进2速度 -200：
  AA 27 04 02 01 FF 38 F8
  偏移量 = -200 (0xFF38)
```

**清零偏移量 — CMD_OFFSET_RESET (0x28)**

```
格式: [电机ID]
ID:  0x11~0x13=直流电机  0x01~0x03=步进电机  0x1F=全部
```

**示例：**
```
清零DCM2偏移量：  AA 28 01 12 3A
清零全部便宜量：  AA 28 01 1F 37
```

### 6.3 偏移量工作流程

```
语音"刀片1启动60%"：
  → CMD_DCM_START      → 偏移量=0，实际=60%

语音"快一点" × 3：
  → CMD_OFFSET_ADJUST  → 偏移量=+15，实际=60%+15%=75%

语音"慢一点"：
  → CMD_OFFSET_ADJUST  → 偏移量=+10，实际=60%+10%=70%

语音"刀片1设为50%"（绝对值）：
  → CMD_DCM_START      → 便宜量清零，实际=50%

语音"停止"：
  → CMD_DCM_STOP       → 停止，偏移量保留（下次启动可继续累加）

语音"清零偏移"：
  → CMD_OFFSET_RESET   → 偏移量=0
```

> **注意：** `CMD_DCM_START` / `CMD_SET_SPEED_DIR` 执行时会**自动清零**对应电机的偏移量，确保绝对值设置的准确性。

# 连接串口

```

## 八、校验和计算示例

### C语言实现

```c
uint8_t Protocol_CalculateChecksum(uint8_t cmd, uint8_t len, uint8_t* data)
{
    uint8_t checksum = cmd ^ len;
    for (uint8_t i = 0; i < len; i++) {
        checksum ^= data[i];
    }
    return checksum;
}
```

### Python实现

```python
def checksum(cmd: int, length: int, data: bytes) -> int:
    cs = cmd ^ length
    for b in data:
        cs ^= b
    return cs
```

## 九、故障排查

### 9.1 收不到ACK响应

**检查项：**
1. 确认语音模块连接到正确的串口（USART1 = PA9/PA10）
2. 确认 STM32 波特率为 115200
3. 确认共地连接（STM32 GND ↔ 语音模块 GND）
4. 检查 TX/RX 是否接反
5. 确认 STM32 程序已烧录最新版本

### 9.2 校验和不匹配

每帧命令都需要重新计算校验和，不能使用旧帧数据。校验和计算示例：

```
帧: AA [CMD] [LEN] [DATA...]
CS = CMD ^ LEN ^ data[0] ^ ... ^ data[len-1]
```

### 9.3 电机不响应

1. 确认驱动器电源接通（DM860H 需要 DC 24~80V）
2. 确认 ENA 使能电平正确（低电平使能）
3. 检查急停按钮是否按下
4. 查看调试串口 USART1 的日志输出

### 9.4 偏移量累积导致速度异常

如果发现电机速度逐渐变快或变慢，说便宜移量已累积。发送清零命令：

```
全部偏移量清零：  AA 28 01 1F 37
```
