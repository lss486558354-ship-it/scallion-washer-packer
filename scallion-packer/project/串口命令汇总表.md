# 串口命令汇总表

本文档与固件一致，依据 `User/protocol/protocol.h`、`protocol.c`（`Protocol_ProcessCommand`）、`process_control.h` / `process_control.c` 整理。若与旧版《串口命令速查表》冲突，**以本表与源码为准**。

---

## 1. 物理串口与协议类型

| 串口 | 典型用途 | 协议 |
|------|----------|------|
| **USART2** | Jetson 等 | `[0xAA][CMD][LEN][DATA…][CHK]`（当前 `USART2_HW_OnRxByte` 仅写入环形缓冲，**中断内未调用** `Protocol_CommChannel_HandleByte`，若需解析需在主循环消费后自行调用） |
| **USART3** | 淘晶驰串口屏 | 同上；**`USART3_HW_OnRxByte` 内已调用** `Protocol_CommChannel_HandleByte(USART3, byte)`，ACK 经 UART3 回屏 |
| **USART1** | 调试打印、HX711 测试任务 | 文本 + 单字节 ASCII 命令（非 0xAA 帧） |
| **UART4** | 飞特总线舵机（SCS/STS） | `FF FF ID LEN INST … CHK`（与0xAA 帧无关） |

`Protocol_CommChannel_HandleByte`：**USART3 接收中断内**逐字节解析 0xAA 帧；ACK 与应答通过 **收到该帧的同一 USART** 回传（`s_current_channel`）。详见工程内 `TJC_HMI_事件与协议对照.md`。

---

## 2. 0xAA 帧格式与校验

-结构：`[0xAA][命令1B][长度 N][N 字节数据][校验 1B]`
- 最大数据长度：`PROTOCOL_MAX_DATA_LEN` = **32**
- 校验：`CHK = 0xAA ^ CMD ^ LEN ^ DATA[0] ^ … ^ DATA[N-1]`（异或，含帧头）

---

## 3. ACK 子协议（部分下行命令的应答）

满足条件时，MCU 发送：

- **ACK 命令字节** = 原命令 + **0x80**
- **数据**固定 2 字节：`[状态][设备/上下文 ID]`

**状态码**（`protocol.c`）：

| 值 | 含义 |
|----|------|
| 0x00 | ACK_OK，成功 |
| 0x01 | ACK_PARAM，参数错误 |
| 0x02 | ACK_UNSUP（定义存在，当前处理中未使用） |
| 0x03 | ACK_BUSY，忙（如工艺流程非 IDLE 时再次 START） |
| 0x04 | ACK_IDLE（定义存在，当前处理中未使用） |

**示例**：收到 `CMD_DCM_START (0x20)` 成功时，应答 `0xA0`，数据 `[0x00][电机ID]`。

**注意**：下列 Jetson 侧命令在 `protocol.c` 中**未调用** `Protocol_SendACK`，上位机不应依赖 ACK：`CMD_SET_MOTOR_SPEED (0x14)`、`CMD_SET_MOTOR_POS (0x15)`、`CMD_SET_SERVO_ANGLE (0x16)`。

## 4.下行命令一览（主机 → STM32）

| CMD | 名称 | 数据区（DATA） | 行为摘要 | 应答 |
|-----|------|----------------|----------|------|
| 0x10 | CMD_START_TASK | 无要求 | `g_system_status = RUNNING` | ACK |
| 0x11 | CMD_STOP_TASK | 无要求 | M1~M3 Stop，状态 IDLE | ACK |
| 0x12 | CMD_PAUSE_TASK | 无要求 | M1~M3 Brake，状态 PAUSED | ACK |
| 0x13 | CMD_RESUME_TASK | 无要求 | 状态 RUNNING | ACK |
| 0x14 | CMD_SET_MOTOR_SPEED | `[电机ID 1~3][速度高][速度低]`，有符号目标速度 | 对应 M1~M3 `Run(speed, DIR_FORWARD)` | **无 ACK** |
| 0x15 | CMD_SET_MOTOR_POS | `[电机ID][位置高][位置低][控制模式]` |仅当 `control_mode==1` 时各电机 `Run(1000, DIR_FORWARD)`（占位逻辑） | **无 ACK** |
| 0x16 | CMD_SET_SERVO_ANGLE | `[舵机ID][角度高][角度低]` | 角度 = `(u16)/10.0` °；**仅 ID 1、2** 映射 `SERVO_1/2`（PWM 舵机） | **无 ACK** |
| 0x17 | CMD_RESET_SYSTEM | 无要求 |停步进+直流，清错误码，状态 READY | ACK |
| 0x18 | CMD_CONFIG_PARAMS | — | **枚举存在，`Protocol_ProcessCommand` 无分支，当前不处理** | — |
| 0x20 | CMD_DCM_START | `[电机ID][占空比0~100][方向可选]` | ID：`0x11/12/13` 单路，`0x1F` 三路同参；方向缺省正转，`0x01` 反转 | ACK |
| 0x21 | CMD_DCM_STOP | `[电机ID]` | `0x1F` 全停，否则0x11~0x13 | ACK |
| 0x22 | CMD_STEPPER_START | `[0x01~0x03][速度高][速度低]` | M1~M3，`DIR_FORWARD` | ACK |
| 0x23 | CMD_STEPPER_STOP | `[0x01~0x03]` | 对应 Stop | ACK |
| 0x24 | CMD_SET_SPEED_DIR | `[设备ID][类型][方向][值高][值低]` | 类型 `0x01` 步进：值=速度，叠加 `g_stepper_offset` 后限幅 1~20000；`0x02` 直流：值=占空比，叠加 `g_dcm_offset` 后 0~100；方向 `0x00` 正转、`0x01` 反转；步进 ID `0x01~03`，直流 `0x11~13` 或 `0x1F` | ACK |
| 0x25 | CMD_STEPPER_DIR | `[0x01~03][方向]` |仅设置 DIR 电平：`0x00` CW、`0x01` CCW | ACK |
| 0x26 | CMD_DCM_DIR | `[0x11~13][方向]` | 写入 `DCM[x].direction`，下次 Run 生效 | ACK |
| 0x27 | CMD_OFFSET_ADJUST | `[设备ID][类型][偏移高][偏移低]` | 类型 `0x01` 步进偏移 ±20000；`0x02` 直流偏移 ±100；有符号 int16 增量 | ACK |
| 0x28 | CMD_OFFSET_RESET | `[设备ID]` | `0x1F` 清零全部步进+直流偏移；`0x11~13` 直流；`0x01~03` 步进 | ACK |
| 0x2F | CMD_ALL_STOP | 无要求 | 步进全停、直流全停、`Process_EmergencyStop`、状态 IDLE | ACK |
| 0x30 | CMD_PROCESS_START | 无要求 |仅 `Process_GetPhase()==IDLE` 时 `Process_Start`；否则 ACK_BUSY | ACK |
| 0x31 | CMD_PROCESS_STOP | 无要求 | `Process_Stop`，状态 IDLE | ACK |
| 0x32 | CMD_PROCESS_PAUSE | 无要求 | `Process_Pause`，状态 PAUSED | ACK |
| 0x33 | CMD_PROCESS_RESUME | 无要求 | `Process_Resume`，状态 RUNNING | ACK |
| 0x34 | CMD_SET_CLEAN_MODE | `[模式]` | 写入 `g_clean_mode` 并 `Process_SetParam(0xF1, mode)` | ACK |
| 0x35 | CMD_SET_PACK_MODE | `[模式]` | 写入 `g_pack_mode` 并 `Process_SetParam(0xF2, mode)` | ACK |
| 0x36 | CMD_SET_PARAM | `[参数ID][值高][值低]` | 见第 6 节 | ACK |
| 0x37 | CMD_GET_STATUS | 无要求 | 见第 5 节（**非 ACK**，直接同 CMD 回 8 字节） | 8B 数据帧 |
| 0x38 | CMD_ULTRASONIC_TEST | 无要求 | `Process_ReadPackDistance()`，3字节测距结果 | 3B 数据帧 |

## 5. 上行 / 应答帧（STM32 → 主机）

### 5.1 周期性 / 主动上报

| CMD | 含义 | 数据长度 | 内容说明（按实现） |
|-----|------|----------|-------------------|
| 0x01 | CMD_SENSOR_DATA | **13** | `[M1_POS_H,L][M2_POS_H,L][M3_POS_H,L][M1_SPD_H,L][M2_SPD_H,L][M3_SPD_H,L][ADC低8位]`；`SensorData_TypeDef` 中舵机角度字段**未编入本帧**（`Protocol_SendSensorData`） |
| 0x04 | CMD_SYSTEM_STATUS | 3 | `[系统状态][错误码][0]` |
| 0x05 | CMD_TASK_COMPLETE | 2 | `[1][0]`（`Protocol_SendTaskComplete`） |
| 0x06 | CMD_ERROR_REPORT | 2 | `[错误码][0]` |

**说明**：`CMD_MOTOR_STATUS (0x02)`、`CMD_LIMIT_SWITCH (0x03)` 在 `protocol.h` 中保留，**当前工程未检索到发送实现**，可视为预留。

### 5.2 CMD_GET_STATUS (0x37) 应答

直接发送 **`CMD=0x37`**，长度 **8**，数据：

| 偏移 | 含义 |
|------|------|
| 0 | `Process_GetPhase()` 主流程阶段 |
| 1 | `Process_GetCleanState()` |
| 2 | `Process_GetPackState()` |
| 3 | `g_clean_mode` |
| 4 | `g_pack_mode` |
| 5 | `g_system_status` |
| 6~7 | 保留0 |

### 5.3 CMD_ULTRASONIC_TEST (0x38) 应答

长度 **3**：`[距离_mm高][距离_mm 低][超时标志]`；`0xFFFF` 距离时标志字节为 `0xFF`，否则 `0x00`。

| ID | 枚举名 | 作用 |

| 0x01 | PARAM_BELT_SPEED_HZ | 同步带速度 Hz |

| 0x03 | PARAM_MEASURE_DELAY_MS | 测距前延时 |
| 0x04 | PARAM_SCREW_TARGET_FIXED | 跳过测距时丝杆固定步数 |
| 0x05 | PARAM_SCREW_SPEED_HZ | 丝杆速度 Hz |
| 0x06 | PARAM_CUTTER_SPEED_DUTY | 切割电机占空比 0~100 |
| 0x07 | PARAM_CUTTING_TIME_MS | 切割持续时间 |
| 0x08 | PARAM_CLEAN_AIR_TIME_MS | 喷气清洗时间 |
| 0x09 | PARAM_CLEAN_WATER_TIME_MS | 水泵清洗时间 |
| 0x0A | PARAM_WEIGHT_TARGET_G | 称重目标 g |
| 0x0B | PARAM_WEIGH_TIMEOUT_MS | 称重超时 |
| 0x0C | PARAM_CONVEYOR_SPEED_HZ | 传送带速度 Hz |
| 0x0D | PARAM_PACK_SPEED_HZ | 打包卷膜速度 Hz |
| 0x0E | PARAM_PACK_WRAP_TIME_MS | 卷膜时间 |
| 0x0F | PARAM_PACK_MIDDLE_DELAY_MS | 打包中间模式延时 |
| 0x10 | PARAM_PACK_HEAD_INTERVAL_MS | 头尾模式间隔 |
| 0x11 | PARAM_PACK_CUT_DELAY_MS | 切断后等待 |
| 0x12 | PARAM_PACK_SERVO_CUT | **打包切断舵机角度**（0~180，工艺用） |
| 0x13 | PARAM_PACK_DIST_HEAD | 葱头阈值 mm |
| 0x14 | PARAM_PACK_DIST_TAIL | 葱尾阈值 mm |
| 0x15 | PARAM_PACK_TIMEOUT_MS | 打包总超时 |
| 0x16 | PARAM_SCREW_LIMIT_DOWN_MS | 丝杆正转限时 ms |
| 0x17 | PARAM_SCREW_LIMIT_UP_MS | 丝杆反转限时 ms |
| 0x18 | PARAM_WEIGHT_STABLE_MS | 称重达标连续保持 ms |
| 0x19 | PARAM_WEIGHT_HYSTERESIS_G | 称重回差 g |

**说明**：`ProcessParams_TypeDef` 中有 `pack_servo_angle_start`（默认复位角），但 **`Process_SetParam` / `Process_GetParam` 未实现对该字段的 ID**，只能通过代码/默认值修改。

**清洗模式**（`CMD_SET_CLEAN_MODE` 或 PARAM0xF1）：`0x00` 未设置 / `0x01` 仅气 / `0x02` 仅水 / `0x03` 气+水。

**打包模式**（`CMD_SET_PACK_MODE` 或 PARAM 0xF2）：`0x00` 未设置 / `0x01` 全流程 / `0x02` 中间 / `0x03` 头尾。

## 7. 舵机相关（汇总）

### 7.1 协议命令 CMD_SET_SERVO_ANGLE (0x16) — PWM 舵机

- **走 0xAA 帧**，**不是**总线舵机。
- 数据：`[舵机ID][角度16位大端]`，角度物理值 = **整型 / 10.0（度）**。
-固件仅处理 **ID = 1 或 2** → `SERVO_1` / `SERVO_2`（`bsp_servo_obj` PWM）。

### 7.2 工艺中的打包舵机角度

- **切断角**：串口改 `CMD_SET_PARAM`，**param_id = 0x12**（`pack_servo_angle_cut`）。
- **复位角**：结构体字段存在，**当前无串口参数 ID**，由默认宏 `DFLT_PACK_SERVO_START` 等初始化。

### 7.3 飞特总线舵机（UART4）

- **不经过** 本协议 0xAA 命令；帧格式见 `User/servo/bsp_servo_serial.h` 头文件注释（`FF FF ID LEN INST … CHK`）。
- 常用寄存器（与头文件一致）：**0x2A** 目标块（位置/时间/速度）、**0x2E** 轮模式目标速度（小端有符号）、**0x28** 扭矩开关；STS 轮模式可配合 `ServoBus_STS_EnterWheelMode`、`ServoBus_STS_WriteSpeedAcc` 等。
- 详细步骤与上位机调参见工程内 **`飞特总线舵机_单片机通讯方法.md`**。

### 7.4 USART1 调试（HX711 任务）

单字节 **ASCII**，**非 0xAA**（`HX711_OnByte`）：

| 字符 | 作用 |
|------|------|
| `T` / `t` | 去皮 |
| `R` / `r` | 清除去皮 |
| `C` / `c` | 打印当前配置 |
